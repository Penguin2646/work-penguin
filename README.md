# 🐧 work-penguin

> 업무 동료 AI **"펭귄"** — 비개발 직원도 자기 업무를 한 단계씩 업그레이드하는 Claude Code 스킬.
> 자비스처럼 **"펭귄"**이라고 부르면 켜집니다.

러쉬코리아 커머셜 플래닝 팀에서 시작한, 사내 업무 업그레이드용 스킬입니다.
"뭘 자동화/개선할 수 있는지조차 모르는" 동료를 위해 만들었습니다.

## 무엇을 하나요?

대화 한 번에 이 **루프**를 돕니다:

1. **발굴** — "어떤 게 고민이세요?" 로 시작해 지금 업무 방식을 인터뷰
2. **진단** — 이 업무가 업계 사다리에서 지금 몇 단계인지, 최상위는 어떤 모습인지
3. **처방** — 한 칸 위로 가는 방법 + 쓰면 좋은 (이미 설치된) 스킬 추천·사용법
4. **제작** — 말로 끝내지 않고 **실제 산출물**을 만들어 줌 (필요하면 여러 에이전트가 병렬로)
5. **측정** — 기존 대비 효과(시간/지표/퀄리티)를 재서 **누적 스코어보드**에 기록, 다음에 실제 효과가 있었는지 추적

**고정된 결과물을 찍어내지 않습니다.** 당신의 질문·수준·니즈에 따라 매번 다르게 움직이고, 도구가 늘수록 똑똑해집니다.

## 설치

Claude Code가 깔려 있어야 합니다. 워크스페이스의 `.claude/skills/` 안에 이 repo를 넣으면 됩니다.

```bash
# 워크스페이스 루트에서
cd .claude/skills
git clone https://github.com/Penguin2646/work-penguin.git
```

또는 Claude Code 채팅창에 이렇게 부탁해도 됩니다:
> "이 깃허브 스킬 설치해줘: https://github.com/Penguin2646/work-penguin"

## 함께 쓰는 도구 — graphify (컴패니언)

펭귄은 진단할 때, 대화가 **실제 업무 폴더**로 향하면 [graphify](https://github.com/safishamsi/graphify)를 꺼내 그 폴더를 지식그래프로 만들어 "지금 상태"를 그림으로 보여줍니다. 미리 안 깔아도 돼요 — **필요해지는 순간 펭귄이 자동 감지해서 설치를 안내**합니다 (PyPI `graphifyy`, 처음 한 번만). graphify가 없어도 펭귄은 대화 인터뷰만으로 진단을 진행합니다(우아한 폴백). _참고: Windows에서 설치 경로가 매우 길면 numpy 의존성이 Long Path 제한에 걸릴 수 있어요 — 짧은 경로에 설치하거나 Long Path를 켜면 됩니다._

## 사용

채팅창에 **펭귄**을 부르세요:

```
펭귄아, 나 매달 보고서 만드는 거 너무 오래 걸려
```

펭귄이 등장해서 같이 미팅을 시작합니다. 🐧

## 구성

```
work-penguin/
├── SKILL.md                  # 펭귄의 두뇌 (5단계 루프 지휘)
├── references/
│   ├── flow-guide.md         # 단계별 인터뷰 질문
│   ├── maturity-ladders.md   # 업무유형별 성숙도 사다리
│   ├── skill-recommend.md    # 설치 스킬 동적 추천 로직
│   └── measurement.md        # 효과 측정법
├── assets/
│   ├── penguin-banner.txt    # 등장 ASCII 펭귄
│   └── view-template.html    # 결과 시각화 웹 뷰 템플릿
├── LICENSE                   # MIT
└── README.md
```

스코어보드는 설치한 사람의 워크스페이스에 `work-penguin/scoreboard.md`로 쌓입니다 (스킬 업데이트해도 안 지워짐).

## 업데이트 (이미 설치한 분)

자동 업데이트는 안 돼요. 설치 폴더에서 **한 줄**이면 최신이 됩니다 (재설치 아님):
```bash
cd .claude/skills/work-penguin
git pull
```
펭귄을 부를 때 새 버전이 있으면 **"🐧 새 버전 있어요 — 받을까요?"** 라고 알려줘요. yes 하면 펭귄이 `git pull`을 대신 해줍니다(또는 위 명령을 안내). 갱신분은 그 다음 호출부터 반영돼요.
