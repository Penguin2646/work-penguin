# maturity-ladders — 업무 성숙도 사다리 (seed)

> 이건 **시드(출발점)**다. 여기 없는 업무 유형이 나오면 펭귄이 같은 4단계 틀(L1 수작업 → L4 시스템)로 **즉석 생성**하고, 가치 있으면 이 파일에 추가 제안한다.
> 사용자에게는 "L2/L3"라는 말보다 **그 단계의 모습**을 쉬운 말로 설명한다.

## 공통 틀

| Lv | 성격 | 한 줄 |
|----|------|-------|
| L1 | 수작업 · 텍스트 | 손으로, 기억과 메모로 |
| L2 | 문서화 · 정형 | 정리된 문서/표로 |
| L3 | 반자동 · 스킬활용 | 템플릿+도구로 반복 제거 |
| L4 | 자동 · 시스템 | 앱/웹/파이프라인이 알아서 |

---

## 📄 보고서 (Reports)

| Lv | 모습 | 예시 도구 |
|----|------|-----------|
| L1 | 메일/메신저로 텍스트 결과 공유 | — |
| L2 | 정리된 문서·PDF로 보고 | md-to-pdf, hwpxskill |
| L3 | 템플릿+스킬로 반복 리포트 자동 구성 | sales-analysis, content-worker, pm-skills |
| L4 | 실시간 대시보드/웹앱, PPT 없는 임원 보고 | dashboard-prd, frontend-design, webapp-prd |

## 📊 데이터 분석 (Data Analysis)

| Lv | 모습 | 예시 도구 |
|----|------|-----------|
| L1 | 엑셀에서 눈으로 보고 수기 계산 | — |
| L2 | 엑셀 정리·피벗, 수동 차트 | excel-to-csv, csv-clean |
| L3 | 스킬로 전처리+분석+예측 반자동 | sales-analysis, product-forecast, analysis-worker |
| L4 | 자동 파이프라인·실시간 모니터링·지식그래프 | graphify, 대시보드 |

## 🗂 기획 · 플랜 (Planning)

| Lv | 모습 | 예시 도구 |
|----|------|-----------|
| L1 | 머릿속/메모로 즉흥 진행 | — |
| L2 | 문서로 구조화(목차·근거) | thinking-partner, idea |
| L3 | 프레임워크+병렬 리서치로 기획 가속 | pm-skills, research-worker, decompose/execute |
| L4 | PRD→구현까지 연결되는 기획 시스템 | dashboard-prd, webapp-prd |

## 🎨 디자인 (Design)

| Lv | 모습 | 예시 도구 |
|----|------|-----------|
| L1 | 기본 템플릿·클립아트로 대충 | — |
| L2 | 가이드 지켜 일관되게 | ui-ux-pro-max(원칙 참고) |
| L3 | 디자인 스킬로 고퀄 산출물 빠르게 | frontend-design, ui-ux-pro-max, taste-skill |
| L4 | 디자인 시스템·재사용 컴포넌트로 양산 | frontend-design + 컴포넌트화 |

## 🔁 반복 수작업 (Repetitive Ops)

| Lv | 모습 | 예시 도구 |
|----|------|-----------|
| L1 | 매번 손으로 복붙·변환 | — |
| L2 | 체크리스트/템플릿으로 실수 줄임 | todo, 템플릿 |
| L3 | 스킬·스크립트로 변환 자동화 | excel-to-csv, pdf-to-md, development-worker |
| L4 | 트리거로 도는 자동 워크플로 | 스크립트/스케줄 + save |

---

**진단 원칙**: 현재 위치를 짚고 → 최상위(L4)를 보여주고 → **바로 위 한 칸**만 구체적으로 제시한다. 입문자에게 L1→L4 점프를 강요하지 않는다.
