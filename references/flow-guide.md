# flow-guide — 5단계 인터뷰 가이드

펭귄이 단계별로 쓰는 질문 스크립트. **그대로 읽지 말고**, 상황에 맞게 골라 쓴다. 한 번에 질문 하나.

## 1단계 — 인사 & 성공기준

- "어떤 게 고민이세요? 요즘 제일 손 많이 가는 업무가 뭐예요?"
- (막막해하면) "괜찮아요. '매주/매달 반복하는데 시간 잡아먹는 일' 하나만 떠올려 볼까요?"
- 성공기준 확인 (쉬운 말로):
  - "이걸 **빨리 끝내고** 싶으세요? (⏱️ 시간)"
  - "아니면 **숫자로 효과를 보여주고** 싶으세요? (📊 지표)"
  - "아니면 **결과물을 더 멋지게** 만들고 싶으세요? (✨ 퀄리티)"
- 여러 개여도 됨. 메모해 둔다.

## 2단계 — 업무 발굴 + 성숙도 진단

현재 방식을 단계별로 캐묻는다:
- "지금은 이걸 어떤 순서로 하세요? 시작부터 끝까지 한번 말씀해 주실래요?"
- "재료(입력)는 어디서 와요? 엑셀? 메일? 누가 줘요?"
- "그걸 받으면 뭘 하세요? (가공/계산/정리)"
- "결과물은 어떤 형태로 나가요? (텍스트/PPT/PDF/엑셀)"
- "얼마나 자주 하세요? 한 번에 시간 얼마나 걸려요?"
- "제일 짜증나거나 막히는 지점은 어디예요?"

→ 들은 걸 **Before 스냅샷**으로 요약해 확인받는다:
> "정리하면: [입력] → [처리] → [산출], 주기 OO, 1회 약 OO분, 막히는 곳은 OO. 맞나요?"

→ **성숙도 진단** (maturity-ladders.md 참조). 쉬운 말로:
> "지금은 거의 손으로 하시는 단계예요(레벨2). 같은 업무를 최상위로 하는 분들은 [최상위 모습]까지 가 있어요. 바로 그 위 칸은 이렇게 넘어갑니다."

## 3단계 — 후보 & 우선순위 + 레벨업 처방

- "제가 보기엔 이렇게 더 잘할 수 있는 길이 몇 개 있어요." → 후보 2~4개
- 각 후보: **무엇을 / 어떻게 / 뭘로(스킬)** 3줄
- skill-recommend.md로 **설치된 스킬 동적 스캔** 후 추천 + 사용법
- "이 중에 **노력 대비 효과**가 제일 좋은 건 OO이에요. 이거부터 해볼까요?"

## 4단계 — 실제 제작

- "그럼 바로 만들어 볼게요." (말로 끝내지 않는다)
- 독립 하위작업 있으면 worker 에이전트 병렬 위임 (SKILL.md 참조)
- 산출물을 사용자에게 보여주고 확인받는다.

## 5단계 — 효과 측정 & 기록

- "원래 OO분 걸리던 게 → 이제 OO분/OO클릭이에요." (1단계 기준으로)
- 📊 기준이면 measurement.md의 정량 추정
- ✨ 기준이면 Before/After 산출물 퀄리티 비교
- 스코어보드에 1줄 기록.
- "다음에 부르시면 이게 **실제로 효과 있었는지** 같이 확인해요." (후속확인 약속)

## 워크숍 모드 변형

- 각 참가자에게 1~2단계를 짧게 → 후보를 한 보드에 모음 → 팀이 노력 대비 효과로 **투표/우선순위** → 1~2건만 4단계 제작 시연.
